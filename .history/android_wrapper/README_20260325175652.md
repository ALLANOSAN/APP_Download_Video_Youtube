# YouTube Downloader Wrapper (Flutter)

## 1. Preparação

1. Instale Flutter: https://flutter.dev/docs/get-started/install
2. Execute `flutter doctor` e corrija avisos.
3. No mesmo PC, rode o backend do projeto:
   ```bash
   cd /mnt/Dados1/APP_Download_Video_Youtube
   export FCM_SERVER_KEY="<sua_server_key>"
   uvicorn src.server.main:app --reload --host 0.0.0.0 --port 8000
   ```
4. Inicie a UI Flet (web):
   ```bash
   flet run mobile/main_flet.py --web
   ```

## 2. Configurar o Flutter wrapper

1. Abra `android_wrapper` no Android Studio ou IDE.
2. No `lib/main.dart`, ajuste:
   - `backendUrl` (IP do servidor, ex: `http://192.168.1.50:8000`)
   - `fletUrl` (ex: `http://192.168.1.50:8555`)

## 3. Setup Firebase

1. No console do Firebase, crie projeto e app Android.
2. Baixe `google-services.json` e coloque em `android/app/`.
3. Edite:
   - `android/build.gradle` com plugin `com.google.gms:google-services`
   - `android/app/build.gradle` adicionando `apply plugin: 'com.google.gms.google-services'`
4. Ajuste `minSdkVersion` >= 21.

## 4. Rodar no dispositivo

- Conecte dispositivo via USB ou emulador.
- `flutter run` (ou `flutter build apk --release`).

## 5. Verificar FCM

1. Confirme token no log:
   - `FCM token: ...`
2. Chame endpoint Backend:
   ```bash
   curl -X POST http://localhost:8000/fcm/send \
     -H "Content-Type: application/json" \
     -d '{"user_id":"mobile_user_1","title":"Teste","body":"OK","data":{}}'
   ```
3. Deverá aparecer notificação no dispositivo.

## 6. Notas

- A lógica de download continua no backend Python, não precisa migrar.
- O wrapper mostra a UI Flet por WebView, então o mesmo fluxo de download funciona.
